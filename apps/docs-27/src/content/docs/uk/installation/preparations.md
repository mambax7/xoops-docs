title: "phpinfo"
---
Цей крок є необов’язковим, але може легко врятувати вас від годин розчарування.

У якості попереднього тестування системи хостингу дуже маленький, але корисний сценарій PHP створюється локально та завантажується в цільову систему.

Сценарій PHP складається лише з одного рядка:
```php
<?php phpinfo();
```
Використовуючи текстовий редактор, створіть файл з назвою _info.php_ з цим одним рядком.

Далі завантажте цей файл у свій веб-корінь.

![Filezilla info.php Завантаження](/xoops-docs/2.7/img/installation/filezilla-01-info.png)

Отримайте доступ до свого сценарію, відкривши його у своєму браузері, тобто перейшовши до `http://example.com/info.php`. Якщо все працює правильно, ви повинні побачити приблизно таку сторінку:

![Приклад phpinfo()](/xoops-docs/2.7/img/installation/php-info.png)

Примітка: деякі служби хостингу можуть вимкнути функцію _phpinfo()_ як міру безпеки. Зазвичай ви отримаєте відповідне повідомлення, якщо це так.

Результати сценарію можуть стати в нагоді для усунення несправностей, тому подумайте про те, щоб зберегти його копію.

Якщо тест спрацює, ви можете приступити до встановлення. Ви повинні видалити сценарій _info.php_ і продовжити встановлення.

Якщо тест провалився, з’ясуйте, чому! Будь-яка проблема, яка заважає цьому простому тесту працювати, **завадить** справжньому встановленню працювати.
