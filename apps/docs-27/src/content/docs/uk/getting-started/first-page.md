# Створення першої сторінки в XOOPS

Дізнайтеся, як створити, відформатувати та опублікувати свій перший вміст у XOOPS.

## Розуміння вмісту XOOPS

### Що таке Page/Post?

У XOOPS контентом керують через модулі. Найпоширеніші типи вмісту:

| Тип | Опис | Випадок використання |
|---|---|---|
| **Сторінка** | Статичний вміст | Про нас, контакти, послуги |
| **Post/Article** | Вміст із міткою часу | Новини, Повідомлення в блозі |
| **Категорія** | Організація змісту | Груповий пов'язаний вміст |
| **Коментар** | Відгуки користувачів | Дозволити взаємодію відвідувачів |

Цей посібник описує створення базового page/article за допомогою стандартного модуля вмісту XOOPS.

## Доступ до редактора вмісту

### З панелі адміністратора

1. Увійдіть до панелі адміністратора: `http://your-domain.com/xoops/admin/`
2. Перейдіть до **Вміст > Сторінки** (або ваш модуль вмісту)
3. Натисніть «Додати нову сторінку» або «Новий допис»

### Інтерфейс (якщо ввімкнено)

Якщо ваш XOOPS налаштовано на створення зовнішнього вмісту:

1. Увійдіть як зареєстрований користувач
2. Перейдіть до свого профілю
3. Знайдіть опцію «Надіслати вміст».
4. Виконайте описані нижче дії

## Покроковий посібник: створення першої сторінки

### Крок 1: Доступ до редактора вмісту

1. На панелі адміністратора натисніть **Вміст > Сторінки**
2. Натисніть **"Додати нову сторінку"** або **"Створити"**
3. Ви побачите редактор вмісту

### Крок 2: Введіть назву сторінки

У полі «Назва» введіть назву вашої сторінки:
```
Title: Welcome to Our Website
```
Рекомендації щодо назв:
- Чіткий і описовий
- Додайте ключові слова, якщо можливо
- 50-60 символів ідеально
- Уникайте ВСІХ ВЕЛИКИХ БУКВ (важко читати)
- Будьте конкретними (не "Сторінка 1")

### Крок 3: Виберіть категорію

Виберіть, де організувати цей вміст:
```
Category: [Dropdown ▼]
```
Варіанти можуть включати:
- Загальний
- Новини
- Блог
- Оголошення
- Послуги

Якщо категорії не існують, попросіть адміністратора створити їх.

### Крок 4: Напишіть свій вміст

Натисніть в області редактора вмісту та введіть текст.

#### Базове форматування тексту

Використовуйте панель інструментів редактора:

| Кнопка | Дія | Результат |
|---|---|---|
| **B** | Жирний | **Жирний текст** |
| *Я* | Курсив | *Курсив* |
| <u>U</u> | Підкреслити | <u>Підкреслений текст</u> |

#### Використання HTML

XOOPS дозволяє використовувати безпечні теги HTML. Загальні приклади:
```html
<!-- Paragraphs -->
<p>This is a paragraph.</p>

<!-- Headings -->
<h1>Main Heading</h1>
<h2>Subheading</h2>

<!-- Lists -->
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
</ul>

<!-- Bold and Italic -->
<strong>Bold text</strong>
<em>Italic text</em>

<!-- Links -->
<a href="https://example.com">Link text</a>

<!-- Line breaks -->
<br>

<!-- Horizontal rule -->
<hr>
```
#### Безпечний HTML Приклади

**Рекомендовані теги:**
- Абзаци: `<p>`, `<br>`
- Заголовки: `<h1>` до `<h6>`
- Текст: `<strong>`, `<em>`, `<u>`
- Списки: `<ul>`, `<ol>`, `<li>`
- Посилання: `<a href="">`
- Блок цитат: `<blockquote>`
- Столи: `<table>`, `<tr>`, `<td>`

**Уникайте цих тегів** (може бути вимкнено з міркувань безпеки):
- Сценарії: `<script>`
- Стилі: `<style>`
- iframes: `<iframe>` (якщо не налаштовано)
- Форми: `<form>`, `<input>`

### Крок 5: Додайте зображення

#### Варіант 1: Вставте зображення URL

Використання редактора:

1. Натисніть кнопку **Вставити зображення** (значок зображення)
2. Введіть зображення URL: `https://example.com/image.jpg`
3. Введіть альтернативний текст: "Опис зображення"
4. Натисніть «Вставити»

Еквівалент HTML:
```html
<img src="https://example.com/image.jpg" alt="Description">
```
#### Варіант 2: Завантажте зображення

1. Спочатку завантажте зображення до XOOPS:
   - Перейдіть до **Content > Media Manager**
   - Завантажте своє зображення
   - Скопіюйте зображення URL

2. У редакторі вмісту вставте за допомогою URL (вказані вище дії)

#### Рекомендації щодо зображення

- Використовуйте відповідні розміри файлів (оптимізуйте зображення)
- Використовуйте описові назви файлів
- Завжди включайте альтернативний текст (доступність)
- Підтримувані формати: JPG, PNG, GIF, WebP
- Рекомендована ширина: 600-800 пікселів для вмісту

### Крок 6: Вставте медіа

#### Вставити відео з YouTube
```html
<iframe width="560" height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  frameborder="0" allowfullscreen>
</iframe>
```
Замініть `VIDEO_ID` на ідентифікатор відео YouTube.

**Щоб знайти ідентифікатор відео YouTube:**
1. Відкрийте відео на YouTube
2. URL це: `https://www.youtube.com/watch?v=VIDEO_ID`
3. Скопіюйте ID (символи після `v=`)

#### Вставити відео з Vimeo
```html
<iframe src="https://player.vimeo.com/video/VIDEO_ID"
  width="640" height="360" frameborder="0"
  allow="autoplay; fullscreen" allowfullscreen>
</iframe>
```
### Крок 7: Додайте мета-опис

У полі «Опис» додайте короткий зміст:
```
Description: Learn how to get started with our website.
This page provides an overview of our services and how we can help you.
```
**Рекомендації щодо метаопису:**
- 150-160 символів
- Додайте основні ключові слова
- Повинен точно підсумовувати зміст
- Використовується в результатах пошуку
- Зробіть це привабливим (користувачі бачать це)

### Крок 8: Налаштуйте параметри публікації

#### Статус публікації

Виберіть статус публікації:
```
Status: ☑ Published
```
Опції:
- **Опубліковано:** Видно для всіх
- **Чернетка:** видима лише для адміністраторів
- **Очікує на розгляд:** Очікує на затвердження
- **Архівовано:** приховано, але збережено

#### Видимість

Укажіть, хто може переглядати цей вміст:
```
Visibility: ☐ Public
           ☐ Registered Users Only
           ☐ Private (Admin Only)
```
#### Дата публікації

Встановити, коли вміст стає видимим:
```
Publish Date: [Date Picker] [Time]
```
Залиште як "Зараз", щоб опублікувати негайно.

#### Дозволити коментарі

Увімкнути або вимкнути коментарі відвідувачів:
```
Allow Comments: ☑ Yes
```
Якщо ввімкнено, відвідувачі можуть додавати відгуки.

### Крок 9: Збережіть свій вміст

Кілька варіантів збереження:
```
[Publish Now]  [Save as Draft]  [Schedule]  [Preview]
```
- **Опублікувати зараз:** зробити видимим негайно
- **Зберегти як чернетку:** Наразі залишити приватним
- **Розклад:** Опублікувати в майбутньому date/time
- **Попередній перегляд:** подивіться, як це виглядає перед збереженням

Натисніть свій вибір:
```
Click [Publish Now]
```
### Крок 10: підтвердьте свою сторінку

Після публікації перевірте вміст:

1. Перейдіть на домашню сторінку свого сайту
2. Перейдіть до області вмісту
3. Знайдіть свою щойно створену сторінку
4. Натисніть, щоб переглянути його
5. Перевірте:
   - [ ] Вміст відображається правильно
   - [ ] З’являються зображення
   - [ ] Форматування виглядає добре
   - [ ] Посилання працюють
   - [ ] Назва та опис правильні

## Керування вашим вмістом

### Редагувати існуючу сторінку

1. Перейдіть до **Вміст > Сторінки**
2. Знайдіть свою сторінку в списку
3. Натисніть **Редагувати** або назву сторінки
4. Внесіть зміни
5. Натисніть **Оновити**

### Видалити сторінку

1. Перейдіть до **Вміст > Сторінки**
2. Знайдіть свою сторінку
3. Натисніть **Видалити**
4. Підтвердьте видалення

### Змінити статус публікації

1. Перейдіть до **Вміст > Сторінки**
2. Знайдіть сторінку, натисніть **Редагувати**
3. Змініть статус у спадному меню
4. Натисніть **Оновити**

## Усунення несправностей створення вмісту

### Вміст не відображається

**Симптом:** опублікована сторінка не відображається на веб-сайті

**Рішення:**
1. Перевірте статус публікації: має бути "Опубліковано"
2. Перевірте дату публікації: має бути поточною або минулою
3. Перевірте видимість: має бути "Для всіх"
4. Очистіть кеш: Адміністратор > Інструменти > Очистити кеш
5. Перевірте дозволи: група користувачів повинна мати доступ

### Не працює форматування

**Симптом:** теги HTML або форматування відображаються як текст

**Рішення:**
1. Переконайтеся, що HTML увімкнено в налаштуваннях модуля
2. Використовуйте правильний синтаксис HTML
3. Закрийте всі теги: `<p>Text</p>`
4. Використовуйте лише дозволені теги
5. Використовуйте сутності HTML: `&lt;` для `<`, `&amp;` для `&`

### Зображення не відображаються

**Симптом:** зображення показують зламаний значок

**Рішення:**
1. Перевірте правильність зображення URL
2. Перевірте наявність файлу зображення
3. Перевірте належні дозволи на зображення
4. Натомість спробуйте завантажити зображення на XOOPS
5. Перевірте зовнішнє блокування (може знадобитися CORS)

### Проблеми з кодуванням символів

**Симптом:** спеціальні символи виглядають як тарабарщина

**Рішення:**
1. Збережіть файл у кодуванні UTF-8
2. Переконайтеся, що кодування сторінки – UTF-8
3. Додайте до головки HTML: `<meta charset="UTF-8">`
4. Уникайте копіювання та вставки з Word (використовуйте звичайний текст)

## Рекомендації щодо роботи з контентом

### Рекомендований процес

1. **Спочатку напишіть у редакторі:** використовуйте адміністративний редактор вмісту
2. **Попередній перегляд перед публікацією:** Натисніть кнопку «Попередній перегляд».
3. **Додати метадані:** Повний заголовок, опис, теги
4. **Спочатку збережіть як чернетку:** збережіть як чернетку, щоб уникнути втрати роботи
5. **Остаточний огляд:** Перечитайте перед публікацією
6. **Опублікувати:** Натисніть Опублікувати, коли будете готові
7. **Перевірте:** Перевірте на сайті
8. **Відредагуйте, якщо потрібно:** Швидко внесіть виправлення

### Контроль версій

Завжди зберігайте резервні копії:

1. **Перед серйозними змінами:** Збережіть як нову версію або резервну копію
2. **Архівуйте старий вміст:** зберігайте неопубліковані версії
3. **Датуйте свої чернетки:** Використовуйте чіткі назви: «Page-Draft-2025-01-28»
