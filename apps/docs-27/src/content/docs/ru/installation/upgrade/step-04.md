## Обновите системный модуль

После применения всех необходимых патчей выбор _Continue_ установит все для обновления модуля **system**. Это очень важный шаг и требуется для надлежащего завершения обновления.

![XOOPS Update System Module](/xoops-docs/2.7/img/installation/upgrade-06-update-system-module.png)

Выберите _Update_, чтобы выполнить обновление системного модуля.

## Обновите другие поставляемые модули XOOPS

XOOPS поставляется с тремя дополнительными модулями - pm (Private Messaging,) profile (User Profile) и protector (Protector). Вы должны выполнить обновление на любом из этих установленных модулей.

![XOOPS Update Other Modules](/xoops-docs/2.7/img/installation/upgrade-07-update-modules.png)

## Обновите другие модули

Вероятно, что существуют обновления для других модулей, которые могут позволить модулям работать лучше при вашем теперь обновленном XOOPS. Вы должны исследовать и применять любые подходящие обновления модулей.

## Просмотрите новые предпочтения закаления файлов cookie

Обновление XOOPS 2.7.0 добавляет два новых предпочтения, которые контролируют, как выпускаются файлы cookie сеанса:

* **`session_cookie_samesite`** — контролирует атрибут файла cookie SameSite. `Lax` - это безопасное значение по умолчанию для большинства сайтов. Используйте `Strict` для максимальной защиты, если ваш сайт не полагается на навигацию по происхождению. `None` уместен только, если вы знаете, что вам это нужно.
* **`session_cookie_secure`** — при включении файл cookie сеанса отправляется только через HTTPS соединения. Включите это, если ваш сайт работает на HTTPS.

Вы можете просмотреть эти параметры в разделе "Параметры системы" → "Предпочтения" → "Основные параметры".

## Проверьте пользовательские темы

Если ваш сайт использует пользовательскую тему, пройдите через переднюю часть и область администратора, чтобы подтвердить, что страницы отображаются правильно. Обновление на Smarty 4 может повлиять на пользовательские шаблоны даже если сканирование предполета пройдено. Если вы видите проблемы с рендерингом, перейдите к [Troubleshooting](ustep-03.md).

## Очистить установку и файлы обновления

Для безопасности удалите эти каталоги из веб-корневой после подтверждения работы обновления:

* `upgrade/` — каталог рабочего процесса обновления
* `install/` — если присутствует, либо как `install/`, либо как переименованный каталог `installremove*`

Оставление их на месте подвергает сценарии обновления и установки для всех, кто может достичь вашего сайта.

## Откройте ваш сайт

Если вы следовали совету отключить ваш сайт, вы должны включить его обратно после того, как определите, что он работает правильно.
