Эта страница собирает информацию о файловой системе и веб-адресе, необходимую XOOPS для работы. Установщик попытается угадать эту информацию на основе URL, который вы используете для доступа к нему, и расположения запущенных скриптов PHP.

После проверки и исправления любых проблем нажмите кнопку "Продолжить" для продолжения.

![XOOPS Installer Path Settings](/xoops-docs/2.7/img/installation/installer-04-02.png)

## Данные, собираемые на этом этапе

### Физические пути XOOPS

#### Физический путь корневого каталога XOOPS

Физический путь к каталогу документов XOOPS (обслуживаемый) БЕЗ косой черты в конце

#### Каталог файлов данных XOOPS

Физический путь к каталогу файлов данных XOOPS (доступный для записи) БЕЗ косой черты в конце. Поместите папку вне корневого каталога документов XOOPS, чтобы сделать её безопасной.

#### Каталог библиотеки XOOPS

Физический путь к каталогу библиотеки XOOPS БЕЗ косой черты в конце. Поместите папку вне корневого каталога документов XOOPS, чтобы сделать её безопасной.

### Веб-адреса

#### Расположение веб-сайта (URL)

Главный URL, который будет использоваться для доступа к вашей установке XOOPS. XOOPS будет использовать это для создания всех URL внутри XOOPS. Если вам нужен сайт _https_, убедитесь, что вы указали это. Если вы хотите или не хотите видеть _www._ в начале доменного имени вашего сайта, убедитесь, что вы указали это как вам нужно.

По умолчанию это будет URL, используемый для запуска установщика.

#### Домен для установки cookies веб-сайта

Домен для установки cookies. Может быть пустым, полным хостом из URL (www.example.com) или зарегистрированным доменом без поддоменов (example.com) для совместного использования поддоменами (www.example.com и blog.example.com.)

## Ошибки

Если какой-либо из введенных путей не может быть найден, будет отображена ошибка. Исправьте любые проблемы перед продолжением.

![XOOPS Installer Path Settings Error](/xoops-docs/2.7/img/installation/installer-04-01.png)

## Справка
