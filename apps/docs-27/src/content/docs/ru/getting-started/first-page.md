# Создание вашей первой страницы в XOOPS

Узнайте, как создавать, форматировать и публиковать свой первый контент в XOOPS.

## Понимание содержимого XOOPS

### Что такое страница/пост?

В XOOPS содержимое управляется через модули. Наиболее распространённые типы содержимого:

| Тип | Описание | Случай использования |
|---|---|---|
| **Страница** | Статическое содержимое | О нас, Контакты, Услуги |
| **Пост/Статья** | Содержимое с меткой времени | Новости, Посты в блоге |
| **Категория** | Организация содержимого | Группировка связанного содержимого |
| **Комментарий** | Обратная связь пользователей | Разрешить взаимодействие посетителей |

Это руководство охватывает создание базовой страницы/статьи, используя модуль содержимого по умолчанию XOOPS.

## Доступ к редактору содержимого

### Из панели администратора

1. Войдите в панель администратора: `http://your-domain.com/xoops/admin/`
2. Перейдите в **Содержимое > Страницы** (или ваш модуль содержимого)
3. Нажмите "Добавить новую страницу" или "Новый пост"

### Фронтенд (если включено)

Если ваш XOOPS настроен на разрешение создания содержимого на фронтенде:

1. Войдите как зарегистрированный пользователь
2. Перейдите на ваш профиль
3. Найдите опцию "Отправить содержимое"
4. Следуйте тем же шагам ниже

## Пошаговое руководство: создание вашей первой страницы

### Шаг 1: Доступ к редактору содержимого

1. В панели администратора нажмите **Содержимое > Страницы**
2. Нажмите **"Добавить новую страницу"** или **"Создать"**
3. Вы увидите редактор содержимого

### Шаг 2: Введите название страницы

В поле "Заголовок" введите имя вашей страницы:

```
Заголовок: Добро пожаловать на наш сайт
```

Лучшие практики для заголовков:
- Ясные и описательные
- Включите ключевые слова, если возможно
- 50-60 символов идеально
- Избегайте ВСЕ ЗАГЛАВНЫЕ (трудно читать)
- Будьте конкретны (не "Страница 1")

### Шаг 3: Выберите категорию

Выберите, где организовать это содержимое:

```
Категория: [Выпадающий список ▼]
```

Опции могут включать:
- Общие
- Новости
- Блог
- Объявления
- Услуги

Если категорий не существует, попросите администратора их создать.

### Шаг 4: Напишите ваше содержимое

Щелкните в области редактора содержимого и введите ваш текст.

#### Базовое форматирование текста

Используйте панель инструментов редактора:

| Кнопка | Действие | Результат |
|---|---|---|
| **B** | Жирный | **Жирный текст** |
| *I* | Курсив | *Курсивный текст* |
| <u>U</u> | Подчеркивание | <u>Подчёркнутый текст</u> |

#### Использование HTML

XOOPS позволяет безопасные теги HTML. Общие примеры:

```html
<!-- Абзацы -->
<p>Это абзац.</p>

<!-- Заголовки -->
<h1>Главный заголовок</h1>
<h2>Подзаголовок</h2>

<!-- Списки -->
<ul>
  <li>Элемент 1</li>
  <li>Элемент 2</li>
  <li>Элемент 3</li>
</ul>

<!-- Жирный и курсив -->
<strong>Жирный текст</strong>
<em>Курсивный текст</em>

<!-- Ссылки -->
<a href="https://example.com">Текст ссылки</a>

<!-- Разрывы строк -->
<br>

<!-- Горизонтальная линия -->
<hr>
```

#### Примеры безопасного HTML

**Рекомендуемые теги:**
- Абзацы: `<p>`, `<br>`
- Заголовки: `<h1>` до `<h6>`
- Текст: `<strong>`, `<em>`, `<u>`
- Списки: `<ul>`, `<ol>`, `<li>`
- Ссылки: `<a href="">`
- Блокировки цитат: `<blockquote>`
- Таблицы: `<table>`, `<tr>`, `<td>`

**Избегайте этих тегов** (может быть отключено для безопасности):
- Скрипты: `<script>`
- Стили: `<style>`
- Iframe: `<iframe>` (если не настроено)
- Формы: `<form>`, `<input>`

### Шаг 5: Добавьте изображения

#### Вариант 1: Вставить URL изображения

Используя редактор:

1. Нажмите кнопку **Вставить изображение** (значок изображения)
2. Введите URL изображения: `https://example.com/image.jpg`
3. Введите альтернативный текст: "Описание изображения"
4. Нажмите "Вставить"

Эквивалент HTML:

```html
<img src="https://example.com/image.jpg" alt="Описание">
```

#### Вариант 2: Загрузить изображение

1. Сначала загрузите изображение в XOOPS:
   - Перейдите в **Содержимое > Менеджер медиа**
   - Загрузите ваше изображение
   - Скопируйте URL изображения

2. В редакторе содержимого вставьте с помощью URL (выше шаги)

#### Лучшие практики для изображений

- Используйте надлежащие размеры файлов (оптимизируйте изображения)
- Используйте описательные имена файлов
- Всегда включайте альтернативный текст (доступность)
- Поддерживаемые форматы: JPG, PNG, GIF, WebP
- Рекомендуемая ширина: 600-800 пикселей для содержимого

### Шаг 6: Встройте медиа

#### Встроить видео с YouTube

```html
<iframe width="560" height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  frameborder="0" allowfullscreen>
</iframe>
```

Замените `VIDEO_ID` на ID видео YouTube.

**Как найти ID видео YouTube:**
1. Откройте видео на YouTube
2. URL: `https://www.youtube.com/watch?v=VIDEO_ID`
3. Скопируйте ID (символы после `v=`)

#### Встроить видео с Vimeo

```html
<iframe src="https://player.vimeo.com/video/VIDEO_ID"
  width="640" height="360" frameborder="0"
  allow="autoplay; fullscreen" allowfullscreen>
</iframe>
```

### Шаг 7: Добавьте мета-описание

В поле "Описание" добавьте краткую сводку:

```
Описание: Узнайте, как начать работу с нашим сайтом.
Эта страница предоставляет обзор наших услуг и того, как мы можем вам помочь.
```

**Лучшие практики мета-описания:**
- 150-160 символов
- Включите основные ключевые слова
- Должно точно резюмировать содержимое
- Используется в результатах поисковых систем
- Сделайте его привлекательным (пользователи видят это)

### Шаг 8: Настройте параметры публикации

#### Статус публикации

Выберите статус публикации:

```
Статус: ☑ Опубликовано
```

Опции:
- **Опубликовано:** Видно общественности
- **Черновик:** Видно только администраторам
- **Ожидает рецензии:** Ожидание утверждения
- **Архивировано:** Скрыто, но сохранено

#### Видимость

Установите, кто может видеть это содержимое:

```
Видимость: ☐ Общественное
           ☐ Только зарегистрированные пользователи
           ☐ Частное (только администратор)
```

#### Дата публикации

Установите, когда содержимое станет видимым:

```
Дата публикации: [Выбор даты] [Время]
```

Оставьте "Сейчас" для немедленной публикации.

#### Разрешить комментарии

Включить или отключить комментарии посетителей:

```
Разрешить комментарии: ☑ Да
```

Если включено, посетители могут добавить обратную связь.

### Шаг 9: Сохраните ваше содержимое

Несколько параметров сохранения:

```
[Опубликовать сейчас] [Сохранить как черновик] [Запланировать] [Просмотр]
```

- **Опубликовать сейчас:** Сделать видимым немедленно
- **Сохранить как черновик:** Держать в частном порядке на данный момент
- **Запланировать:** Опубликовать в будущую дату/время
- **Просмотр:** Посмотрите, как это выглядит перед сохранением

Нажмите ваш выбор:

```
Нажмите [Опубликовать сейчас]
```

### Шаг 10: Проверьте вашу страницу

После публикации проверьте ваше содержимое:

1. Перейдите на главную страницу вашего веб-сайта
2. Перейдите в область вашего содержимого
3. Ищите вашу недавно созданную страницу
4. Нажмите, чтобы просмотреть
5. Проверьте:
   - [ ] Содержимое отображается правильно
   - [ ] Изображения появляются
   - [ ] Форматирование выглядит хорошо
   - [ ] Ссылки работают
   - [ ] Заголовок и описание правильны

## Управление вашим содержимым

### Редактировать существующую страницу

1. Перейдите в **Содержимое > Страницы**
2. Найдите вашу страницу в списке
3. Нажмите **Редактировать** или название страницы
4. Внесите изменения
5. Нажмите **Обновить**

### Удалить страницу

1. Перейдите в **Содержимое > Страницы**
2. Найдите вашу страницу
3. Нажмите **Удалить**
4. Подтвердите удаление

### Изменить статус публикации

1. Перейдите в **Содержимое > Страницы**
2. Найдите страницу, нажмите **Редактировать**
3. Измените статус в выпадающем списке
4. Нажмите **Обновить**

## Решение проблем при создании содержимого

### Содержимое не отображается

**Симптом:** Опубликованная страница не показывается на сайте

**Решение:**
1. Проверьте статус публикации: Должен быть "Опубликовано"
2. Проверьте дату публикации: Должна быть текущей или прошедшей
3. Проверьте видимость: Должна быть "Общественное"
4. Очистите кэш: Администратор > Инструменты > Очистить кэш
5. Проверьте разрешения: Группа пользователей должна иметь доступ

### Форматирование не работает

**Симптом:** Теги HTML или форматирование отображаются как текст

**Решение:**
1. Проверьте, включен ли HTML в параметрах модуля
2. Используйте правильный синтаксис HTML
3. Закройте все теги: `<p>Текст</p>`
4. Используйте только разрешённые теги
5. Используйте HTML-сущности: `&lt;` для `<`, `&amp;` для `&`

### Изображения не отображаются

**Симптом:** Изображения показывают значок поломанного изображения

**Решение:**
1. Проверьте, правильный ли URL изображения
2. Проверьте существование файла изображения
3. Проверьте правильность разрешений на изображение
4. Попробуйте загрузить изображение в XOOPS
5. Проверьте внешнее блокирование (может потребоваться CORS)

### Проблемы с кодировкой символов

**Симптом:** Специальные символы отображаются как иероглифы

**Решение:**
1. Сохраните файл с кодировкой UTF-8
2. Убедитесь, что кодировка страницы UTF-8
3. Добавьте в голову HTML: `<meta charset="UTF-8">`
4. Избегайте копирования из Word (используйте простой текст)

## Лучшие практики рабочего процесса содержимого

### Рекомендуемый процесс

1. **Написать в редакторе:** Используйте редактор содержимого администратора
2. **Просмотреть перед публикацией:** Нажмите кнопку Просмотр
3. **Добавить метаданные:** Заполните заголовок, описание, теги
4. **Сохранить как черновик:** Сохраните как черновик, чтобы избежать потери работы
5. **Финальный обзор:** Перечитайте перед публикацией
6. **Опубликовать:** Нажмите Опубликовать, когда готово
7. **Проверить:** Проверить на живом сайте
8. **Отредактировать при необходимости:** Быстро исправить ошибки

### Управление версиями

Всегда держите резервные копии:

1. **Перед крупными изменениями:** Сохраните как новую версию или резервную копию
2. **Архивируйте старое содержимое:** Держите неопубликованные версии
3. **Датируйте ваши черновики:** Используйте чёткие имена: "Page-Draft-2025-01-28"
