# ניהול משתמשים ב-XOOPS

למד כיצד ליצור חשבונות משתמש, לארגן משתמשים לקבוצות ולנהל הרשאות ב-XOOPS.

## סקירה כללית של ניהול משתמשים

XOOPS מספק ניהול משתמשים מקיף עם:

```
Users > Accounts
├── Individual users
├── User profiles
├── Registration requests
└── Online users

Users > Groups
├── User groups/roles
├── Group permissions
└── Group membership

System > Permissions
├── Module access
├── Content access
├── Function permissions
└── Group capabilities
```

## גישה לניהול משתמשים

### ניווט בלוח הניהול

1. היכנס למנהל מערכת: `http://your-domain.com/xoops/admin/`
2. לחץ על **משתמשים** בסרגל הצד השמאלי
3. בחר מבין האפשרויות:
   - **משתמשים:** נהל חשבונות בודדים
   - **קבוצות:** נהל קבוצות משתמשים
   - **משתמשים מקוונים:** ראה משתמשים פעילים כרגע
   - **בקשות משתמש:** עיבוד בקשות רישום

## הבנת תפקידי משתמש

XOOPS מגיע עם תפקידי משתמש מוגדרים מראש:

| קבוצה | תפקיד | יכולות | מקרה שימוש |
|---|---|---|---|
| **מנהלי אתרים** | מנהל | שליטה מלאה באתר | מנהלים ראשיים |
| **מנהלים** | מנהל | גישת מנהל מוגבלת | משתמשים מהימנים |
| **מנחים** | בקרת תוכן | אשר תוכן | מנהלי קהילה |
| **עורכים** | יצירת תוכן | Create/edit תוכן | צוות תוכן |
| **רשום** | חבר | פוסט, תגובה, פרופיל | משתמשים קבועים |
| **אנונימי** | מבקר | קריאה בלבד | משתמשים לא מחוברים |

## יצירת חשבונות משתמש

### שיטה 1: מנהל יוצר משתמש

**שלב 1: גישה ליצירת משתמש**

1. עבור אל **משתמשים > משתמשים**
2. לחץ על **"הוסף משתמש חדש"** או **"צור משתמש"**

**שלב 2: הזן פרטי משתמש**

מלא פרטי משתמש:

```
Username: [4+ characters, letters/numbers/underscore only]
Example: john_smith

Email Address: [Valid email address]
Example: john@example.com

Password: [Strong password]
Example: MyStr0ng!Pass2025

Confirm Password: [Repeat password]
Example: MyStr0ng!Pass2025

Real Name: [User's full name]
Example: John Smith

URL: [Optional user website]
Example: https://johnsmith.com

Signature: [Optional forum signature]
Example: "Happy XOOPS user!"
```

**שלב 3: הגדר את הגדרות המשתמש**

```
User Status: ☑ Active
             ☐ Inactive
             ☐ Pending Approval

User Groups:
☑ Registered Users
☐ Webmasters
☐ Admins
☐ Moderators
```

**שלב 4: אפשרויות נוספות**

```
Notify User: ☑ Send welcome email
Allow Avatar: ☑ Yes
User Theme: [Default theme]
Show Email: ☐ Public / ☑ Private
```

**שלב 5: צור חשבון**

לחץ על **"הוסף משתמש"** או **"צור"**

אישור:
```
User created successfully!
Username: john_smith
Email: john@example.com
Groups: Registered Users
```

### שיטה 2: רישום עצמי של משתמש

אפשר למשתמשים להירשם בעצמם:

**לוח ניהול > מערכת > העדפות > הגדרות משתמש**

```
Allow User Registration: ☑ Yes

Registration Type:
☐ Instant (Approve automatically)
☑ Email Verification (Email confirmation)
☐ Admin Approval (You approve each)

Send Verification Email: ☑ Yes
```

לאחר מכן:
1. משתמשים מבקרים בדף ההרשמה
2. מלא מידע בסיסי
3. אמת דוא"ל או המתן לאישור
4. החשבון הופעל

## ניהול חשבונות משתמש

### הצג את כל המשתמשים

**מיקום:** משתמשים > משתמשים

מציג רשימת משתמשים עם:
- שם משתמש
- כתובת אימייל
- תאריך רישום
- כניסה אחרונה
- מצב משתמש (Active/Inactive)
- חברות בקבוצה

### ערוך חשבון משתמש

1. ברשימת המשתמשים, לחץ על שם משתמש
2. שנה כל שדה:
   - כתובת אימייל
   - סיסמה
   - שם אמיתי
   - קבוצות משתמשים
   - סטטוס

3. לחץ על **"שמור"** או **"עדכן"**

### שנה את סיסמת המשתמש

1. לחץ על משתמש ברשימה
2. גלול לקטע "שנה סיסמה".
3. הזן סיסמה חדשה
4. אשר את הסיסמה
5. לחץ על **"שנה סיסמה"**

המשתמש ישתמש בסיסמה חדשה בכניסה הבאה.

### Deactivate/Suspend משתמש

השבת זמנית את החשבון ללא מחיקה:

1. לחץ על משתמש ברשימה
2. הגדר את **סטטוס משתמש** ל"לא פעיל"
3. לחץ על **"שמור"**

המשתמש לא יכול להיכנס בזמן שהוא לא פעיל.

### הפעל מחדש את המשתמש

1. לחץ על משתמש ברשימה
2. הגדר את **סטטוס משתמש** ל"פעיל"
3. לחץ על **"שמור"**

המשתמש יכול להיכנס שוב.

### מחק חשבון משתמש

הסר משתמש לצמיתות:

1. לחץ על משתמש ברשימה
2. גלול למטה
3. לחץ על **"מחק משתמש"**
4. אשר: "מחק את המשתמש ואת כל הנתונים?"
5. לחץ על **"כן"**

**אזהרה:** המחיקה היא לצמיתות!

### הצג פרופיל משתמש

ראה פרטי פרופיל משתמש:

1. לחץ על שם המשתמש ברשימת המשתמשים
2. סקור את פרטי הפרופיל:
   - שם אמיתי
   - אימייל
   - אתר אינטרנט
   - תאריך הצטרפות
   - כניסה אחרונה
   - ביוגרפיה של משתמש
   - אווטאר
   - Posts/contributions

## הבנת קבוצות משתמשים

### קבוצות משתמש ברירת מחדל

XOOPS כולל קבוצות ברירת מחדל:

| קבוצה | מטרה | מיוחד | ערוך |
|---|---|---|---|
| **אנונימי** | משתמשים לא מחוברים | תוקן | לא |
| **משתמשים רשומים** | חברים קבועים | ברירת מחדל | כן |
| **מנהלי אתרים** | מנהלי אתרים | אדמין | כן |
| **מנהלים** | מנהלים מוגבלים | אדמין | כן |
| **מנחים** | מנהלי תוכן | מותאם אישית | כן |

### צור קבוצה מותאמת אישית

צור קבוצה לתפקיד ספציפי:

**מיקום:** משתמשים > קבוצות

1. לחץ על **"הוסף קבוצה חדשה"**
2. הזן את פרטי הקבוצה:

```
Group Name: Content Editors
Group Description: Users who can create and edit content

Display Group: ☑ Yes (Show in member profiles)
Group Type: ☑ Regular / ☐ Admin
```

3. לחץ על **"צור קבוצה"**

### נהל חברות בקבוצה

הקצה משתמשים לקבוצות:

**אפשרות א': מרשימת המשתמשים**

1. עבור אל **משתמשים > משתמשים**
2. לחץ על משתמש
3. קבוצות Check/uncheck בסעיף "קבוצות משתמשים".
4. לחץ על **"שמור"**

**אפשרות ב': מקבוצות**

1. עבור אל **משתמשים > קבוצות**
2. לחץ על שם הקבוצה
3. רשימת חברים View/edit
4. הוסף או הסר משתמשים
5. לחץ על **"שמור"**

### ערוך את מאפייני הקבוצה

התאם אישית את הגדרות הקבוצה:

1. עבור אל **משתמשים > קבוצות**
2. לחץ על שם הקבוצה
3. שנה:
   - שם הקבוצה
   - תיאור הקבוצה
   - קבוצת תצוגה (show/hide)
   - סוג קבוצה
4. לחץ על **"שמור"**

## הרשאות משתמש

### הבנת הרשאות

שלוש רמות הרשאה:

| רמה | היקף | דוגמה |
|---|---|---|
| **גישה למודול** | יכול מודול see/use | יכול לגשת למודול הפורום |
| **הרשאות תוכן** | יכול להציג תוכן ספציפי | יכול לקרוא חדשות שפורסמו |
| **הרשאות פונקציה** | יכול לבצע פעולות | יכול לכתוב תגובות |

### הגדר גישה למודול

**מיקום:** מערכת > הרשאות

הגבל אילו קבוצות יכולות לגשת לכל מודול:

```
Module: News

Admin Access:
☑ Webmasters
☑ Admins
☐ Moderators
☐ Registered Users
☐ Anonymous

User Access:
☐ Webmasters
☐ Admins
☑ Moderators
☑ Registered Users
☑ Anonymous
```

לחץ על **"שמור"** כדי להגיש בקשה.

### הגדר הרשאות תוכן

שליטה בגישה לתוכן ספציפי:

דוגמה - כתבת חדשות:
```
View Permission:
☑ All groups can read

Post Permission:
☑ Registered Users
☑ Content Editors
☐ Anonymous

Moderate Comments:
☑ Moderators required
```

### שיטות מומלצות להרשאה

```
Public Content (News, Pages):
├── View: All groups
├── Post: Registered Users + Editors
└── Moderate: Admins + Moderators

Community (Forum, Comments):
├── View: All groups
├── Post: Registered Users
└── Moderate: Moderators + Admins

Admin Tools:
├── View: Webmasters + Admins only
├── Configure: Webmasters only
└── Delete: Webmasters only
```

## ניהול רישום משתמשים

### טיפול בבקשות רישום

אם "אישור מנהל" מופעל:

1. עבור אל **משתמשים > בקשות משתמשים**
2. הצג רישומים ממתינים:
   - שם משתמש
   - אימייל
   - תאריך רישום
   - סטטוס בקשה

3. לכל בקשה:
   - לחץ לסקירה
   - לחץ על **"אישור"** כדי להפעיל
   - לחץ על **"דחה"** כדי לדחות

### שלח דוא"ל רישום

שלח שוב דוא"ל welcome/verification:

1. עבור אל **משתמשים > משתמשים**
2. לחץ על משתמש
3. לחץ על **"שלח אימייל"** או **"שלח אימות מחדש"**
4. דואר אלקטרוני נשלח למשתמש

## משימות נפוצות לניהול משתמשים

### צור משתמש מנהל

1. צור משתמש חדש (שלבים למעלה)
2. הקצה לקבוצת **מנהלי אתרים** או **מנהלים**
3. הענק הרשאות במערכת > הרשאות
4. ודא שהגישה למנהל מערכת פועלת

### צור מנחה

1. צור משתמש חדש
2. הקצה לקבוצת **מנחים**
3. הגדר הרשאות לניהול מודולים ספציפיים
4. המשתמש יכול לאשר תוכן, לנהל הערות

### הגדר עורכי תוכן

1. צור קבוצת **עורכי תוכן**
2. צור משתמשים, הקצה לקבוצה
3. הענק הרשאות ל:
   - דפי Create/edit
   - Create/edit פוסטים
   - הערות מתונות
4. הגבל את הגישה לפאנל הניהול

### אפס סיסמה שכחת

המשתמש שכח את הסיסמה שלו:

1. עבור אל **משתמשים > משתמשים**
2. מצא משתמש
3. לחץ על שם משתמש
4. לחץ על **"איפוס סיסמה"** או ערוך את שדה הסיסמה
5. הגדר סיסמה זמנית
6. הודע למשתמש (שלח דוא"ל)
7. משתמש נכנס, משנה סיסמה

### ייבוא בכמות גדולה של משתמשים

ייבוא רשימת משתמשים (מתקדם):

לוחות אירוח רבים מספקים כלים ל:
1. הכן קובץ CSV עם נתוני משתמש
2. העלה דרך פאנל ניהול
3. יצירת חשבונות המוני

או השתמש ב-script/plugin מותאם אישית עבור יבוא.

4. לשמור על מדיניות הפרטיות
5. יומן פעילויות עיבוד נתונים

## פתרון בעיות משתמש

### המשתמש לא יכול להתחבר

**בעיה:** המשתמש שכח סיסמה או לא יכול לגשת לחשבון

**פתרון:**
1. ודא שחשבון המשתמש הוא "פעיל"
2. אפס סיסמה:
   - ניהול > משתמשים > מצא משתמש
   - הגדר סיסמה זמנית חדשה
   - שלח למשתמש באמצעות דואר אלקטרוני
3. נקה את המשתמש cookies/cache
4. בדוק אם החשבון אינו נעול

### רישום המשתמש תקוע

**בעיה:** המשתמש לא יכול להשלים את הרישום

**פתרון:**
1. רישום צ'קים מותר:
   - ניהול > מערכת > העדפות > הגדרות משתמש
   - אפשר רישום
2. בדוק את הגדרות הדוא"ל
3. אם נדרש אימות דוא"ל:
   - שלח שוב דוא"ל אימות
   - בדוק את תיקיית הספאם
4. הורד את דרישות הסיסמה אם מחמירות מדי

### חשבונות כפולים

**בעיה:** למשתמש יש מספר חשבונות

**פתרון:**
1. זהה חשבונות כפולים ברשימת המשתמשים
2. שמור על חשבון ראשי
3. מיזוג נתונים במידת האפשר
4. מחק חשבונות כפולים
5. הפעל את "מניעת דוא"ל כפול" בהגדרות

## רשימת רשימות לניהול משתמשים

להגדרה ראשונית:

- [ ] הגדר את סוג רישום המשתמש (instant/email/admin)
- [ ] צור קבוצות משתמשים נדרשות
- [ ] הגדר הרשאות קבוצה
- [ ] הגדר מדיניות סיסמאות
- [ ] אפשר פרופילי משתמש
- [ ] הגדר הודעות דוא"ל
- [ ] הגדר אפשרויות דמות משתמש
- [ ] תהליך רישום למבחן
- [ ] צור חשבונות בדיקה
- [ ] ודא שההרשאות פועלות
- [ ] מבנה קבוצת מסמכים
- [ ] תכנן הצטרפות משתמשים

## השלבים הבאים

לאחר הגדרת משתמשים:

1. התקן מודולים שמשתמשים צריכים
2. צור תוכן למשתמשים
3. חשבונות משתמש מאובטחים
4. חקור תכונות אדמין נוספות
5. הגדר הגדרות כלל המערכת
