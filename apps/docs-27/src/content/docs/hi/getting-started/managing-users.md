# XOOPS में उपयोगकर्ताओं को प्रबंधित करना

उपयोगकर्ता खाते बनाना, उपयोगकर्ताओं को समूहों में व्यवस्थित करना और XOOPS में अनुमतियाँ प्रबंधित करना सीखें।

## उपयोगकर्ता प्रबंधन अवलोकन

XOOPS व्यापक उपयोगकर्ता प्रबंधन प्रदान करता है:

```
Users > Accounts
├── Individual users
├── User profiles
├── Registration requests
└── Online users

Users > Groups
├── User groups/roles
├── Group permissions
└── Group membership

System > Permissions
├── Module access
├── Content access
├── Function permissions
└── Group capabilities
```

## उपयोगकर्ता प्रबंधन तक पहुँचना

### व्यवस्थापक पैनल नेविगेशन

1. एडमिन में लॉग इन करें: `http://your-domain.com/xoops/admin/`
2. बाएं साइडबार में **उपयोगकर्ता** पर क्लिक करें
3. विकल्पों में से चयन करें:
   - **उपयोगकर्ता:** व्यक्तिगत खाते प्रबंधित करें
   - **समूह:** उपयोगकर्ता समूह प्रबंधित करें
   - **ऑनलाइन उपयोगकर्ता:** वर्तमान में सक्रिय उपयोगकर्ता देखें
   - **उपयोगकर्ता अनुरोध:** प्रक्रिया पंजीकरण अनुरोध

## उपयोगकर्ता भूमिकाओं को समझना

XOOPS पूर्वनिर्धारित उपयोगकर्ता भूमिकाओं के साथ आता है:

| समूह | भूमिका | क्षमताएं | केस का प्रयोग करें |
|---|---|---|---|
| **वेबमास्टर्स** | प्रशासक | पूर्ण साइट नियंत्रण | मुख्य व्यवस्थापक |
| **प्रशासक** | प्रशासक | सीमित व्यवस्थापक पहुंच | विश्वसनीय उपयोगकर्ता |
| **मॉडरेटर** | सामग्री नियंत्रण | सामग्री स्वीकृत करें | सामुदायिक प्रबंधक |
| **संपादक** | सामग्री निर्माण | सामग्री बनाएं/संपादित करें | सामग्री कर्मचारी |
| **पंजीकृत** | सदस्य | पोस्ट, टिप्पणी, प्रोफ़ाइल | नियमित उपयोगकर्ता |
| **गुमनाम** | आगंतुक | केवल पढ़ें | गैर-लॉग-इन उपयोगकर्ता |

## उपयोगकर्ता खाते बनाना

### विधि 1: व्यवस्थापक उपयोगकर्ता बनाता है

**चरण 1: उपयोगकर्ता निर्माण तक पहुंचें**

1. **उपयोगकर्ता > उपयोगकर्ता** पर जाएं
2. **"नया उपयोगकर्ता जोड़ें"** या **"उपयोगकर्ता बनाएं"** पर क्लिक करें

**चरण 2: उपयोगकर्ता जानकारी दर्ज करें**

उपयोगकर्ता विवरण भरें:

```
Username: [4+ characters, letters/numbers/underscore only]
Example: john_smith

Email Address: [Valid email address]
Example: john@example.com

Password: [Strong password]
Example: MyStr0ng!Pass2025

Confirm Password: [Repeat password]
Example: MyStr0ng!Pass2025

Real Name: [User's full name]
Example: John Smith

URL: [Optional user website]
Example: https://johnsmith.com

Signature: [Optional forum signature]
Example: "Happy XOOPS user!"
```

**चरण 3: उपयोगकर्ता सेटिंग्स कॉन्फ़िगर करें**

```
User Status: ☑ Active
             ☐ Inactive
             ☐ Pending Approval

User Groups:
☑ Registered Users
☐ Webmasters
☐ Admins
☐ Moderators
```

**चरण 4: अतिरिक्त विकल्प**

```
Notify User: ☑ Send welcome email
Allow Avatar: ☑ Yes
User Theme: [Default theme]
Show Email: ☐ Public / ☑ Private
```

**चरण 5: खाता बनाएं**

**"उपयोगकर्ता जोड़ें"** या **"बनाएँ"** पर क्लिक करें

पुष्टि:
```
User created successfully!
Username: john_smith
Email: john@example.com
Groups: Registered Users
```

### विधि 2: उपयोगकर्ता स्व-पंजीकरण

उपयोगकर्ताओं को स्वयं को पंजीकृत करने की अनुमति दें:

**एडमिन पैनल > सिस्टम > प्राथमिकताएँ > उपयोगकर्ता सेटिंग्स**

```
Allow User Registration: ☑ Yes

Registration Type:
☐ Instant (Approve automatically)
☑ Email Verification (Email confirmation)
☐ Admin Approval (You approve each)

Send Verification Email: ☑ Yes
```

फिर:
1. उपयोगकर्ता पंजीकरण पृष्ठ पर जाएँ
2. बुनियादी जानकारी भरें
3. ईमेल सत्यापित करें या अनुमोदन की प्रतीक्षा करें
4. खाता सक्रिय

## उपयोगकर्ता खातों का प्रबंधन करना

### सभी उपयोगकर्ताओं को देखें

**स्थान:** उपयोगकर्ता > उपयोगकर्ता

उपयोगकर्ता सूची को इसके साथ दिखाता है:
- उपयोक्तानाम
- ईमेल पता
- पंजीकरण की तारीख
- अंतिम लॉगिन
- उपयोगकर्ता स्थिति (सक्रिय/निष्क्रिय)
- समूह सदस्यता

### उपयोगकर्ता खाता संपादित करें

1. उपयोगकर्ता सूची में, उपयोगकर्ता नाम पर क्लिक करें
2. किसी भी फ़ील्ड को संशोधित करें:
   - ईमेल पता
   - पासवर्ड
   - असली नाम
   - उपयोगकर्ता समूह
   - स्थिति

3. **"सहेजें"** या **"अपडेट"** पर क्लिक करें

### उपयोगकर्ता पासवर्ड बदलें

1. सूची में उपयोगकर्ता पर क्लिक करें
2. "पासवर्ड बदलें" अनुभाग तक स्क्रॉल करें
3. नया पासवर्ड डालें
4. पासवर्ड की पुष्टि करें
5. **"पासवर्ड बदलें"** पर क्लिक करें

उपयोगकर्ता अगले लॉगिन पर नए पासवर्ड का उपयोग करेगा।

### उपयोगकर्ता को निष्क्रिय/निलंबित करें

खाते को बिना हटाए अस्थायी रूप से अक्षम करें:

1. सूची में उपयोगकर्ता पर क्लिक करें
2. **उपयोगकर्ता स्थिति** को "निष्क्रिय" पर सेट करें
3. **"सहेजें"** पर क्लिक करें

उपयोगकर्ता निष्क्रिय रहते हुए लॉग इन नहीं कर सकता.

### उपयोगकर्ता को पुनः सक्रिय करें

1. सूची में उपयोगकर्ता पर क्लिक करें
2. **उपयोगकर्ता स्थिति** को "सक्रिय" पर सेट करें
3. **"सहेजें"** पर क्लिक करें

उपयोगकर्ता दोबारा लॉग इन कर सकता है.

### उपयोगकर्ता खाता हटाएं

उपयोगकर्ता को स्थायी रूप से हटाएँ:

1. सूची में उपयोगकर्ता पर क्लिक करें
2. नीचे स्क्रॉल करें
3. **"उपयोगकर्ता हटाएं"** पर क्लिक करें
4. पुष्टि करें: "उपयोगकर्ता और सभी डेटा हटाएं?"
5. **"हाँ"** पर क्लिक करें

**चेतावनी:** विलोपन स्थायी है!

### उपयोगकर्ता प्रोफ़ाइल देखें

उपयोगकर्ता प्रोफ़ाइल विवरण देखें:

1. उपयोगकर्ता सूची में उपयोगकर्ता नाम पर क्लिक करें
2. प्रोफ़ाइल जानकारी की समीक्षा करें:
   - असली नाम
   - ईमेल
   - वेबसाइट
   - शामिल होने की तिथि
   - अंतिम लॉगिन
   - उपयोगकर्ता जीवनी
   - अवतार
   - पोस्ट/योगदान

## उपयोगकर्ता समूहों को समझना

### डिफ़ॉल्ट उपयोगकर्ता समूह

XOOPS में डिफ़ॉल्ट समूह शामिल हैं:| समूह | उद्देश्य | विशेष | संपादित करें |
|---|---|---|---|
| **गुमनाम** | गैर-लॉग-इन उपयोगकर्ता | निश्चित | नहीं |
| **पंजीकृत उपयोगकर्ता** | नियमित सदस्य | डिफ़ॉल्ट | हाँ |
| **वेबमास्टर्स** | साइट प्रशासक | एडमिन | हाँ |
| **प्रशासक** | सीमित व्यवस्थापक | एडमिन | हाँ |
| **मॉडरेटर** | सामग्री मॉडरेटर | कस्टम | हाँ |

### कस्टम ग्रुप बनाएं

विशिष्ट भूमिका के लिए समूह बनाएं:

**स्थान:** उपयोगकर्ता > समूह

1. **"नया समूह जोड़ें"** पर क्लिक करें
2. समूह विवरण दर्ज करें:

```
Group Name: Content Editors
Group Description: Users who can create and edit content

Display Group: ☑ Yes (Show in member profiles)
Group Type: ☑ Regular / ☐ Admin
```

3. **"समूह बनाएं"** पर क्लिक करें

### समूह सदस्यता प्रबंधित करें

उपयोगकर्ताओं को समूहों में असाइन करें:

**विकल्प ए: उपयोगकर्ता सूची से**

1. **उपयोगकर्ता > उपयोगकर्ता** पर जाएं
2. उपयोगकर्ता पर क्लिक करें
3. "उपयोगकर्ता समूह" अनुभाग में समूहों को चेक/अनचेक करें
4. **"सहेजें"** पर क्लिक करें

**विकल्प बी: समूहों से**

1. **उपयोगकर्ता > समूह** पर जाएँ
2. समूह नाम पर क्लिक करें
3. सदस्य सूची देखें/संपादित करें
4. उपयोगकर्ताओं को जोड़ें या हटाएँ
5. **"सहेजें"** पर क्लिक करें

### समूह गुण संपादित करें

समूह सेटिंग अनुकूलित करें:

1. **उपयोगकर्ता > समूह** पर जाएँ
2. समूह नाम पर क्लिक करें
3. संशोधित करें:
   - समूह का नाम
   - समूह विवरण
   - प्रदर्शन समूह (दिखाएँ/छिपाएँ)
   - समूह प्रकार
4. **"सहेजें"** पर क्लिक करें

## उपयोगकर्ता अनुमतियाँ

### अनुमतियाँ समझना

तीन अनुमति स्तर:

| स्तर | दायरा | उदाहरण |
|---|---|---|
| **मॉड्यूल एक्सेस** | मॉड्यूल देख/उपयोग कर सकते हैं | फोरम मॉड्यूल तक पहुंच सकते हैं |
| **सामग्री अनुमतियाँ** | विशिष्ट सामग्री देख सकते हैं | प्रकाशित समाचार पढ़ सकते हैं |
| **फ़ंक्शन अनुमतियाँ** | क्रियाएं कर सकते हैं | टिप्पणियाँ पोस्ट कर सकते हैं |

### मॉड्यूल एक्सेस कॉन्फ़िगर करें

**स्थान:** सिस्टम > अनुमतियाँ

प्रतिबंधित करें कि कौन से समूह प्रत्येक मॉड्यूल तक पहुंच सकते हैं:

```
Module: News

Admin Access:
☑ Webmasters
☑ Admins
☐ Moderators
☐ Registered Users
☐ Anonymous

User Access:
☐ Webmasters
☐ Admins
☑ Moderators
☑ Registered Users
☑ Anonymous
```

आवेदन करने के लिए **"सहेजें"** पर क्लिक करें।

### सामग्री अनुमतियाँ सेट करें

विशिष्ट सामग्री तक पहुंच नियंत्रित करें:

उदाहरण - समाचार लेख:
```
View Permission:
☑ All groups can read

Post Permission:
☑ Registered Users
☑ Content Editors
☐ Anonymous

Moderate Comments:
☑ Moderators required
```

### अनुमति सर्वोत्तम प्रथाएँ

```
Public Content (News, Pages):
├── View: All groups
├── Post: Registered Users + Editors
└── Moderate: Admins + Moderators

Community (Forum, Comments):
├── View: All groups
├── Post: Registered Users
└── Moderate: Moderators + Admins

Admin Tools:
├── View: Webmasters + Admins only
├── Configure: Webmasters only
└── Delete: Webmasters only
```

## उपयोगकर्ता पंजीकरण प्रबंधन

### पंजीकरण अनुरोध संभालें

यदि "व्यवस्थापक अनुमोदन" सक्षम है:

1. **उपयोगकर्ता > उपयोगकर्ता अनुरोध** पर जाएं
2. लंबित पंजीकरण देखें:
   - उपयोक्तानाम
   - ईमेल
   - पंजीकरण की तारीख
   - अनुरोध स्थिति

3. प्रत्येक अनुरोध के लिए:
   - समीक्षा के लिए क्लिक करें
   - सक्रिय करने के लिए **"स्वीकृत करें"** पर क्लिक करें
   - इनकार करने के लिए **"अस्वीकार करें"** पर क्लिक करें

### पंजीकरण ईमेल भेजें

स्वागत/सत्यापन ईमेल पुनः भेजें:

1. **उपयोगकर्ता > उपयोगकर्ता** पर जाएं
2. उपयोगकर्ता पर क्लिक करें
3. **"ईमेल भेजें"** या **"सत्यापन पुनः भेजें"** पर क्लिक करें
4. उपयोगकर्ता को ईमेल भेजा गया

## ऑनलाइन उपयोगकर्ताओं की निगरानी

### वर्तमान में ऑनलाइन उपयोगकर्ता देखें

सक्रिय साइट विज़िटर ट्रैक करें:

**स्थान:** उपयोगकर्ता > ऑनलाइन उपयोगकर्ता

दिखाता है:
- वर्तमान ऑनलाइन उपयोगकर्ता
- अतिथि आगंतुकों की गिनती होती है
- अंतिम गतिविधि का समय
-आईपी पता
- ब्राउज़िंग स्थान

### उपयोगकर्ता गतिविधि की निगरानी करें

उपयोगकर्ता के व्यवहार को समझें:

```
Active Users: 12
Registered: 8
Anonymous: 4

Recent Activity:
- User1 - Forum post (2 min ago)
- User2 - Comment (5 min ago)
- User3 - Page view (8 min ago)
```

## उपयोगकर्ता प्रोफ़ाइल अनुकूलन

### उपयोगकर्ता प्रोफ़ाइल सक्षम करें

उपयोगकर्ता प्रोफ़ाइल विकल्प कॉन्फ़िगर करें:

**व्यवस्थापक > सिस्टम > प्राथमिकताएँ > उपयोगकर्ता सेटिंग्स**

```
Allow User Profiles: ☑ Yes
Show Member List: ☑ Yes
Users Can Edit Profile: ☑ Yes
Show User Avatar: ☑ Yes
Show Last Online: ☑ Yes
Show Email Address: ☐ Yes / ☑ No
```

### प्रोफ़ाइल फ़ील्ड

कॉन्फ़िगर करें कि उपयोगकर्ता प्रोफ़ाइल में क्या जोड़ सकते हैं:

उदाहरण प्रोफ़ाइल फ़ील्ड:
- असली नाम
- वेबसाइट URL
- जीवनी
- स्थान
- अवतार (चित्र)
- हस्ताक्षर
- रुचियाँ
- सोशल मीडिया लिंक

मॉड्यूल सेटिंग्स में अनुकूलित करें।

### दो-कारक प्रमाणीकरण सक्षम करें

उन्नत सुरक्षा विकल्प (यदि उपलब्ध हो):

**व्यवस्थापक > उपयोगकर्ता > सेटिंग्स**

```
Two-Factor Authentication: ☑ Enabled

Methods:
☑ Email
☑ SMS
```

## सामान्य उपयोगकर्ता प्रबंधन कार्य

### व्यवस्थापक उपयोगकर्ता बनाएँ

1. नया उपयोगकर्ता बनाएं (ऊपर दिए गए चरण)
2. **वेबमास्टर्स** या **एडमिन** समूह को असाइन करें
3. सिस्टम > अनुमतियाँ में अनुमतियाँ प्रदान करें
4. व्यवस्थापक पहुंच कार्यों को सत्यापित करें

### मॉडरेटर बनाएं

1. नया उपयोगकर्ता बनाएं
2. **मॉडरेटर** समूह को असाइन करें
3. विशिष्ट मॉड्यूल को मॉडरेट करने के लिए अनुमतियाँ कॉन्फ़िगर करें
4. उपयोगकर्ता सामग्री को स्वीकृत कर सकता है, टिप्पणियाँ प्रबंधित कर सकता है

### सेटअप सामग्री संपादक

1. **सामग्री संपादक** समूह बनाएं
2. उपयोगकर्ता बनाएं, समूह को असाइन करें
3. इन्हें अनुमतियाँ प्रदान करें:
   - पेज बनाएं/संपादित करें
   - पोस्ट बनाएं/संपादित करें
   - संयत टिप्पणियाँ
4. व्यवस्थापक पैनल पहुंच प्रतिबंधित करें

### भूले हुए पासवर्ड को रीसेट करें

उपयोगकर्ता अपना पासवर्ड भूल गए:

1. **उपयोगकर्ता > उपयोगकर्ता** पर जाएं
2. उपयोगकर्ता खोजें
3. उपयोगकर्ता नाम पर क्लिक करें
4. **"पासवर्ड रीसेट करें"** पर क्लिक करें या पासवर्ड फ़ील्ड संपादित करें
5. अस्थायी पासवर्ड सेट करें
6. उपयोगकर्ता को सूचित करें (ईमेल भेजें)
7. उपयोगकर्ता लॉग इन करता है, पासवर्ड बदलता है

### थोक आयात उपयोगकर्ता

उपयोगकर्ता सूची आयात करें (उन्नत):

कई होस्टिंग पैनल निम्न के लिए उपकरण प्रदान करते हैं:
1. उपयोगकर्ता डेटा के साथ CSV फ़ाइल तैयार करें
2. व्यवस्थापक पैनल के माध्यम से अपलोड करें
3. बड़े पैमाने पर खाते बनाएं

या आयात के लिए कस्टम स्क्रिप्ट/प्लगइन का उपयोग करें।

## उपयोगकर्ता समस्याओं का निवारण

### उपयोगकर्ता लॉग इन नहीं कर सकता

**समस्या:** उपयोगकर्ता पासवर्ड भूल गया है या खाते तक नहीं पहुंच पा रहा है

**समाधान:**
1. सत्यापित करें कि उपयोगकर्ता खाता "सक्रिय" है
2. पासवर्ड रीसेट करें:
   - व्यवस्थापक > उपयोगकर्ता > उपयोगकर्ता ढूंढें
   - नया अस्थायी पासवर्ड सेट करें
   - उपयोगकर्ता को ईमेल के माध्यम से भेजें
3. उपयोगकर्ता कुकीज़/कैश साफ़ करें
4. जांचें कि खाता लॉक तो नहीं है

### उपयोगकर्ता पंजीकरण अटक गया

**समस्या:** उपयोगकर्ता पंजीकरण पूरा नहीं कर सकता

**समाधान:**
1. चेक पंजीकरण की अनुमति है:
   - एडमिन > सिस्टम > प्राथमिकताएँ > उपयोगकर्ता सेटिंग्स
   - पंजीकरण सक्षम करें
2. ईमेल सेटिंग कार्य की जांच करें
3. यदि ईमेल सत्यापन आवश्यक हो:
   - सत्यापन ईमेल पुनः भेजें
   - स्पैम फ़ोल्डर की जाँच करें
4. यदि बहुत सख्त हो तो पासवर्ड आवश्यकताएँ कम करें

### डुप्लिकेट खाते

**समस्या:** उपयोगकर्ता के पास एकाधिक खाते हैं

**समाधान:**
1. उपयोगकर्ता सूची में डुप्लिकेट खातों की पहचान करें
2. प्राथमिक खाता रखें
3. यदि संभव हो तो डेटा मर्ज करें
4. डुप्लिकेट खाते हटाएं
5. सेटिंग्स में "डुप्लिकेट ईमेल रोकें" सक्षम करें
