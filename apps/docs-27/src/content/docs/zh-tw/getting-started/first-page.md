# 在 XOOPS 中建立第一個頁面

了解如何在 XOOPS 中建立、格式化和發佈您的第一篇內容。

## 了解 XOOPS 內容

### 什麼是頁面/貼文？

在 XOOPS 中，內容透過模組進行管理。最常見的內容類型是：

| 類型 | 描述 | 使用案例 |
|---|---|---|
| **頁面** | 靜態內容 | 關於我們、聯絡方式、服務 |
| **貼文/文章** | 帶時間戳記的內容 | 新聞、部落格文章 |
| **類別** | 內容組織 | 分組相關內容 |
| **評論** | 使用者意見反應 | 允許訪客互動 |

本指南涵蓋使用 XOOPS 預設內容模組建立基本頁面/文章。

## 存取內容編輯器

### 從管理面板

1. 登入管理面板：`http://your-domain.com/xoops/admin/`
2. 導覽至 **Content > Pages**（或您的內容模組）
3. 按一下「新增頁面」或「新增貼文」

### 前端（如果已啟用）

如果您的 XOOPS 已組態為允許前端內容建立：

1. 以註冊使用者身分登入
2. 移至您的個人檔案
3. 尋找「提交內容」選項
4. 遵循下面的相同步驟

## 逐步指南：建立您的第一個頁面

### 步驟 1：存取內容編輯器

1. 在管理面板中，按一下 **Content > Pages**
2. 按一下 **「新增頁面」** 或 **「建立」**
3. 您會看到內容編輯器

### 步驟 2：輸入頁面標題

在「標題」欄位中，輸入您的頁面名稱：

```
標題：歡迎使用我們的網站
```

標題最佳做法：
- 清晰且具有描述性
- 如果可能，包含關鍵字
- 50-60 個字元最佳
- 避免全部大寫（難以閱讀）
- 具體化（不是「頁面 1」）

### 步驟 3：選擇類別

選擇組織此內容的位置：

```
類別：[下拉式功能表 ▼]
```

選項可能包括：
- 一般
- 新聞
- 部落格
- 公告
- 服務

如果類別不存在，請要求管理員建立它們。

### 步驟 4：撰寫內容

按一下內容編輯器區域並輸入您的文字。

#### 基本文字格式

使用編輯器工具列：

| 按鈕 | 動作 | 結果 |
|---|---|---|
| **B** | 粗體 | **粗體文字** |
| *I* | 斜體 | *斜體文字* |
| <u>U</u> | 底線 | <u>加底線的文字</u> |

#### 使用 HTML

XOOPS 允許安全的 HTML 標籤。常見範例：

```html
<!-- 段落 -->
<p>這是一個段落。</p>

<!-- 標題 -->
<h1>主標題</h1>
<h2>副標題</h2>

<!-- 清單 -->
<ul>
  <li>項目 1</li>
  <li>項目 2</li>
  <li>項目 3</li>
</ul>

<!-- 粗體和斜體 -->
<strong>粗體文字</strong>
<em>斜體文字</em>

<!-- 連結 -->
<a href="https://example.com">連結文字</a>

<!-- 換行符號 -->
<br>

<!-- 水平線 -->
<hr>
```

#### 安全 HTML 範例

**建議的標籤：**
- 段落：`<p>`、`<br>`
- 標題：`<h1>` 至 `<h6>`
- 文字：`<strong>`、`<em>`、`<u>`
- 清單：`<ul>`、`<ol>`、`<li>`
- 連結：`<a href="">`
- 引用：`<blockquote>`
- 表格：`<table>`、`<tr>`、`<td>`

**避免這些標籤**（可能因安全原因而被停用）：
- 指令碼：`<script>`
- 樣式：`<style>`
- Iframe：`<iframe>`（除非已組態）
- 表單：`<form>`、`<input>`

### 步驟 5：新增影像

#### 選項 1：插入影像 URL

使用編輯器：

1. 按一下 **插入影像** 按鈕（影像圖示）
2. 輸入影像 URL：`https://example.com/image.jpg`
3. 輸入替代文字：「影像描述」
4. 按一下「插入」

HTML 等效項：

```html
<img src="https://example.com/image.jpg" alt="描述">
```

#### 選項 2：上傳影像

1. 先上傳影像到 XOOPS：
   - 移至 **Content > Media Manager**
   - 上傳您的影像
   - 複製影像 URL

2. 在內容編輯器中，使用 URL 插入（上面的步驟）

#### 影像最佳做法

- 使用適當的檔案大小（最佳化影像）
- 使用描述性檔案名稱
- 始終包含替代文字（存取性）
- 支援的格式：JPG、PNG、GIF、WebP
- 建議寬度：內容 600-800 像素

### 步驟 6：嵌入媒體

#### 從 YouTube 嵌入影片

```html
<iframe width="560" height="315"
  src="https://www.youtube.com/embed/VIDEO_ID"
  frameborder="0" allowfullscreen>
</iframe>
```

將 `VIDEO_ID` 替換為 YouTube 影片 ID。

**尋找 YouTube 影片 ID：**
1. 在 YouTube 上開啟影片
2. URL 是：`https://www.youtube.com/watch?v=VIDEO_ID`
3. 複製 ID（`v=` 之後的字元）

#### 從 Vimeo 嵌入影片

```html
<iframe src="https://player.vimeo.com/video/VIDEO_ID"
  width="640" height="360" frameborder="0"
  allow="autoplay; fullscreen" allowfullscreen>
</iframe>
```

### 步驟 7：新增中繼描述

在「描述」欄位中，新增簡要摘要：

```
描述：了解如何開始使用我們的網站。
此頁面概述了我們的服務以及我們如何幫助您。
```

**中繼描述最佳做法：**
- 150-160 個字元
- 自然包括主要關鍵字
- 應準確總結內容
- 用於搜尋引擎結果
- 讓人信服（使用者看到這個）

### 步驟 8：組態發佈選項

#### 發佈狀態

選擇發佈狀態：

```
狀態：☑ 已發佈
```

選項：
- **已發佈：** 對公眾可見
- **草稿：** 僅對管理員可見
- **待審核：** 等待核准
- **已封存：** 已隱藏但已保留

#### 可見性

設定誰可以看到此內容：

```
可見性：☐ 公開
       ☐ 僅限註冊使用者
       ☐ 私人（僅限管理員）
```

#### 發佈日期

設定內容變得可見的時間：

```
發佈日期：[日期選擇器] [時間]
```

保留為「現在」可立即發佈。

#### 允許評論

啟用或停用訪客評論：

```
允許評論：☑ 是
```

如果啟用，訪客可以新增意見反應。

### 步驟 9：儲存內容

多個儲存選項：

```
[立即發佈]  [儲存為草稿]  [排程]  [預覽]
```

- **立即發佈：** 立即顯示
- **儲存為草稿：** 目前保持私人
- **排程：** 在未來的日期/時間發佈
- **預覽：** 在儲存前查看其外觀

按一下您的選擇：

```
按一下 [立即發佈]
```

### 步驟 10：驗證您的頁面

發佈後，驗證您的內容：

1. 移至您的網站首頁
2. 導覽至您的內容區域
3. 尋找您新建立的頁面
4. 按一下以檢視
5. 檢查：
   - [ ] 內容顯示正確
   - [ ] 影像出現
   - [ ] 格式看起來不錯
   - [ ] 連結有效
   - [ ] 標題和描述正確


按一下按鈕以無需 HTML 的格式化文字。

### 插入代碼區塊

顯示代碼範例：


將資料組織在表格中：


## 內容的 SEO 最佳做法

針對搜尋引擎最佳化您的內容：

### 標題
- 包含主要關鍵字
- 50-60 個字元
- 每個頁面唯一

### 中繼描述
- 自然包括關鍵字
- 150-160 個字元
- 令人信服且準確

### 內容
- 自然撰寫，避免關鍵字堆砌
- 適當使用標題 (h2, h3)
- 包含指向其他頁面的內部連結
- 在所有影像上使用替代文字
- 文章目標為 300 個字以上

### URL 結構
- 保持 URL 簡短且具有描述性
- 使用連字號分隔文字
- 避免特殊字元
- 範例：`/about-our-company`

## 內容建立疑難排解

### 內容未出現

**症狀：** 已發佈的頁面不顯示在網站上

**解決方案：**
1. 檢查發佈狀態：應為「已發佈」
2. 檢查發佈日期：應為目前或過去
3. 檢查可見性：應為「公開」
4. 清除快取：Admin > Tools > Clear Cache
5. 檢查權限：使用者群組必須有存取權

### 格式化不起作用

**症狀：** HTML 標籤或格式化顯示為文字

**解決方案：**
1. 驗證在模組設定中啟用了 HTML
2. 使用正確的 HTML 語法
3. 關閉所有標籤：`<p>文字</p>`
4. 僅使用允許的標籤
5. 使用 HTML 實體：`&lt;` 用於 `<`，`&amp;` 用於 `&`

### 影像未顯示

**症狀：** 影像顯示損壞的圖示

**解決方案：**
1. 驗證影像 URL 正確
2. 檢查影像檔案是否存在
3. 驗證影像的適當權限
4. 嘗試改為上傳影像到 XOOPS
5. 檢查是否有外部封鎖（可能需要 CORS）

### 字元編碼問題

**症狀：** 特殊字元顯示為亂碼

**解決方案：**
1. 將檔案儲存為 UTF-8 編碼
2. 確保頁面字集為 UTF-8
3. 新增至 HTML head：`<meta charset="UTF-8">`
4. 避免從 Word 複製貼上（使用純文字）

## 內容工作流程最佳做法

### 建議的流程

1. **首先在編輯器中撰寫：** 使用管理內容編輯器
2. **發佈前預覽：** 按一下預覽按鈕
3. **新增中繼資料：** 填寫標題、描述、標籤
4. **先儲存為草稿：** 儲存為草稿以避免遺失工作
5. **最終審查：** 在發佈前重新閱讀
6. **發佈：** 準備好時按一下「發佈」
7. **驗證：** 在即時網站上檢查
8. **如果需要編輯：** 快速進行更正

### 版本控制

始終保留備份：

1. **進行重大變更前：** 儲存為新版本或備份
2. **封存舊內容：** 保留未發佈的版本
3. **為草稿加上日期：** 使用清晰的命名：「Page-Draft-2025-01-28」

## 發佈多個頁面

建立內容策略：

```
首頁
├── 關於我們
├── 服務
│   ├── 服務 1
│   ├── 服務 2
│   └── 服務 3
├── 部落格
│   ├── 文章 1
│   ├── 文章 2
│   └── 文章 3
├── 聯絡方式
└── 常見問題
```

建立頁面以遵循此結構。
