# إدارة المستخدمين في XOOPS

تعلم كيفية إنشاء حسابات المستخدمين وتنظيم المستخدمين إلى مجموعات وإدارة الأذونات في XOOPS.

## نظرة عامة على إدارة المستخدمين

يوفر XOOPS إدارة مستخدمين شاملة مع:

```
المستخدمون > الحسابات
├── المستخدمون الفرديون
├── ملفات تعريف المستخدم
├── طلبات التسجيل
└── المستخدمون الحاليون

المستخدمون > المجموعات
├── مجموعات/أدوار المستخدمين
├── أذونات المجموعة
└── عضوية المجموعة

النظام > الأذونات
├── وصول الوحدة
├── وصول المحتوى
├── أذونات الوظائف
└── إمكانيات المجموعة
```

## الوصول إلى إدارة المستخدمين

### التنقل في لوحة المسؤول

1. تسجيل الدخول إلى المسؤول: `http://your-domain.com/xoops/admin/`
2. انقر على **المستخدمون** في الشريط الجانبي الأيسر
3. اختر من الخيارات:
   - **المستخدمون:** إدارة الحسابات الفردية
   - **المجموعات:** إدارة مجموعات المستخدمين
   - **المستخدمون الحاليون:** انظر المستخدمين النشطين حالياً
   - **طلبات المستخدمين:** عملية طلبات التسجيل

## فهم أدوار المستخدمين

يأتي XOOPS مع أدوار مستخدمين محددة مسبقاً:

| المجموعة | الدور | الإمكانيات | حالة الاستخدام |
|---|---|---|---|
| **صاحب الموقع** | مسؤول | التحكم الكامل بالموقع | المسؤولون الرئيسيون |
| **المسؤولون** | مسؤول | وصول مسؤول محدود | المستخدمون الموثوقون |
| **المشرفون** | التحكم في المحتوى | الموافقة على المحتوى | مديرو المجتمع |
| **المحررون** | إنشاء المحتوى | إنشاء/تحرير المحتوى | موظفو المحتوى |
| **مسجلون** | عضو | نشر وتعليق وملف شخصي | المستخدمون العاديون |
| **مجهولون** | زائر | قراءة فقط | المستخدمون غير المسجلين |

## إنشاء حسابات المستخدمين

### الطريقة 1: ينشئ المسؤول المستخدم

**الخطوة 1: الوصول إلى إنشاء المستخدم**

1. اذهب إلى **المستخدمون > المستخدمون**
2. انقر على **"إضافة مستخدم جديد"** أو **"إنشاء مستخدم"**

**الخطوة 2: أدخل معلومات المستخدم**

ملء تفاصيل المستخدم:

```
اسم المستخدم: [4+ أحرف، أحرف/أرقام/شرطة سفلية فقط]
مثال: john_smith

عنوان البريد الإلكتروني: [عنوان بريد إلكتروني صحيح]
مثال: john@example.com

كلمة المرور: [كلمة مرور قوية]
مثال: MyStr0ng!Pass2025

تأكيد كلمة المرور: [كرر كلمة المرور]
مثال: MyStr0ng!Pass2025

الاسم الحقيقي: [اسم المستخدم الكامل]
مثال: جون سميث

عنوان URL: [موقع ويب اختياري للمستخدم]
مثال: https://johnsmith.com

التوقيع: [توقيع منتدى اختياري]
مثال: "مستخدم XOOPS سعيد!"
```

**الخطوة 3: تكوين إعدادات المستخدم**

```
حالة المستخدم: ☑ نشط
             ☐ غير نشط
             ☐ في انتظار الموافقة

مجموعات المستخدمين:
☑ مستخدمون مسجلون
☐ صاحب الموقع
☐ مسؤولون
☐ مشرفون
```

**الخطوة 4: خيارات إضافية**

```
إخطار المستخدم: ☑ إرسال بريد ترحيب
السماح بالصورة الرمزية: ☑ نعم
موضوع المستخدم: [الموضوع الافتراضي]
إظهار البريد الإلكتروني: ☐ عام / ☑ خاص
```

**الخطوة 5: إنشاء الحساب**

انقر على **"إضافة مستخدم"** أو **"إنشاء"**

تأكيد:
```
تم إنشاء المستخدم بنجاح!
اسم المستخدم: john_smith
البريد الإلكتروني: john@example.com
المجموعات: مستخدمون مسجلون
```

### الطريقة 2: تسجيل المستخدم الذاتي

اسمح للمستخدمين بتسجيل أنفسهم:

**لوحة المسؤول > النظام > التفضيلات > إعدادات المستخدم**

```
السماح بتسجيل المستخدمين: ☑ نعم

نوع التسجيل:
☐ فوري (الموافقة تلقائية)
☑ التحقق بالبريد الإلكتروني (تأكيد البريد الإلكتروني)
☐ موافقة المسؤول (أنت توافق على كل واحد)

إرسال بريد التحقق: ☑ نعم
```

ثم:
1. يزور المستخدمون صفحة التسجيل
2. ملء المعلومات الأساسية
3. التحقق من البريد الإلكتروني أو انتظار الموافقة
4. تفعيل الحساب

## إدارة حسابات المستخدمين

### عرض جميع المستخدمين

**الموقع:** المستخدمون > المستخدمون

يعرض قائمة المستخدمين مع:
- اسم المستخدم
- عنوان البريد الإلكتروني
- تاريخ التسجيل
- آخر تسجيل دخول
- حالة المستخدم (نشط/غير نشط)
- عضوية المجموعة

### تحرير حساب المستخدم

1. في قائمة المستخدمين، انقر على اسم المستخدم
2. عدّل أي حقل:
   - عنوان البريد الإلكتروني
   - كلمة المرور
   - الاسم الحقيقي
   - مجموعات المستخدمين
   - الحالة

3. انقر على **"حفظ"** أو **"تحديث"**

### تغيير كلمة مرور المستخدم

1. انقر على المستخدم في القائمة
2. انتقل إلى قسم "تغيير كلمة المرور"
3. أدخل كلمة مرور جديدة
4. أكد كلمة المرور
5. انقر على **"تغيير كلمة المرور"**

سيستخدم المستخدم كلمة المرور الجديدة عند تسجيل الدخول التالي.

### تعطيل/تعليق المستخدم

عطّل الحساب مؤقتاً بدون حذف:

1. انقر على المستخدم في القائمة
2. اضبط **حالة المستخدم** على "غير نشط"
3. انقر على **"حفظ"**

لا يمكن للمستخدم تسجيل الدخول أثناء عدم النشاط.

### إعادة تفعيل المستخدم

1. انقر على المستخدم في القائمة
2. اضبط **حالة المستخدم** على "نشط"
3. انقر على **"حفظ"**

يمكن للمستخدم تسجيل الدخول مرة أخرى.

### حذف حساب المستخدم

أزل المستخدم بشكل دائم:

1. انقر على المستخدم في القائمة
2. انتقل إلى الأسفل
3. انقر على **"حذف المستخدم"**
4. تأكيد: "حذف المستخدم وجميع البيانات؟"
5. انقر على **"نعم"**

**تحذير:** الحذف دائم!

### عرض ملف تعريف المستخدم

انظر تفاصيل ملف تعريف المستخدم:

1. انقر على اسم المستخدم في قائمة المستخدمين
2. راجع معلومات الملف الشخصي:
   - الاسم الحقيقي
   - البريد الإلكتروني
   - موقع الويب
   - تاريخ الانضمام
   - آخر تسجيل دخول
   - السيرة الذاتية للمستخدم
   - الصورة الرمزية
   - المنشورات/الإسهامات

## فهم مجموعات المستخدمين

### مجموعات افتراضية

يتضمن XOOPS مجموعات افتراضية:

| المجموعة | الغرض | خاص | تحرير |
|---|---|---|---|
| **مجهول** | المستخدمون غير المسجلين | ثابت | لا |
| **مستخدمون مسجلون** | الأعضاء العاديون | افتراضي | نعم |
| **صاحب الموقع** | مسؤولو الموقع | إدارة | نعم |
| **مسؤولون** | مسؤولون محدودون | إدارة | نعم |
| **مشرفون** | معدلو المحتوى | مخصص | نعم |

### إنشاء مجموعة مخصصة

أنشئ مجموعة لدور محدد:

**الموقع:** المستخدمون > المجموعات

1. انقر على **"إضافة مجموعة جديدة"**
2. أدخل تفاصيل المجموعة:

```
اسم المجموعة: محررو المحتوى
وصف المجموعة: المستخدمون الذين يمكنهم إنشاء وتحرير المحتوى

عرض المجموعة: ☑ نعم (عرض في ملفات تعريف الأعضاء)
نوع المجموعة: ☑ عادي / ☐ إدارة
```

3. انقر على **"إنشاء مجموعة"**

### إدارة عضوية المجموعة

عيّن المستخدمين إلى مجموعات:

**الخيار أ: من قائمة المستخدمين**

1. اذهب إلى **المستخدمون > المستخدمون**
2. انقر على المستخدم
3. حدد/أزل تحديد المجموعات في قسم "مجموعات المستخدمين"
4. انقر على **"حفظ"**

**الخيار ب: من المجموعات**

1. اذهب إلى **المستخدمون > المجموعات**
2. انقر على اسم المجموعة
3. عرض/تحرير قائمة الأعضاء
4. أضف أو أزل المستخدمين
5. انقر على **"حفظ"**

### تحرير خصائص المجموعة

خصص إعدادات المجموعة:

1. اذهب إلى **المستخدمون > المجموعات**
2. انقر على اسم المجموعة
3. عدّل:
   - اسم المجموعة
   - وصف المجموعة
   - عرض المجموعة (عرض/إخفاء)
   - نوع المجموعة
4. انقر على **"حفظ"**

## أذونات المستخدمين

### فهم الأذونات

ثلاثة مستويات أذونات:

| المستوى | النطاق | مثال |
|---|---|---|
| **وصول الوحدة** | يمكن رؤية/استخدام الوحدة | يمكن الوصول إلى وحدة المنتدى |
| **أذونات المحتوى** | يمكن عرض محتوى محدد | يمكن قراءة الأخبار المنشورة |
| **أذونات الوظائف** | يمكن تنفيذ الإجراءات | يمكن نشر التعليقات |

### تكوين وصول الوحدة

**الموقع:** النظام > الأذونات

قيّد المجموعات التي يمكنها الوصول إلى كل وحدة:

```
الوحدة: أخبار

وصول المسؤول:
☑ صاحب الموقع
☑ مسؤولون
☐ مشرفون
☐ مستخدمون مسجلون
☐ مجهول

وصول المستخدم:
☐ صاحب الموقع
☐ مسؤولون
☑ مشرفون
☑ مستخدمون مسجلون
☑ مجهول
```

انقر على **"حفظ"** للتطبيق.

### تعيين أذونات المحتوى

التحكم في الوصول إلى محتوى محدد:

مثال - مقالة أخبار:
```
إذن العرض:
☑ جميع المجموعات يمكنها القراءة

إذن النشر:
☑ مستخدمون مسجلون
☑ محررو محتوى
☐ مجهول

معتدل التعليقات:
☑ المشرفون مطلوبون
```

### أفضل ممارسات الأذونات

```
محتوى عام (أخبار، صفحات):
├── العرض: جميع المجموعات
├── النشر: مستخدمون مسجلون + محررون
└── الإشراف: مسؤولون + مشرفون

المجتمع (منتدى، تعليقات):
├── العرض: جميع المجموعات
├── النشر: مستخدمون مسجلون
└── الإشراف: مشرفون + مسؤولون

أدوات المسؤول:
├── العرض: صاحب الموقع + مسؤولون فقط
├── تكوين: صاحب الموقع فقط
└── حذف: صاحب الموقع فقط
```

## إدارة طلبات التسجيل

### التعامل مع طلبات التسجيل

إذا كانت "موافقة المسؤول" مفعلة:

1. اذهب إلى **المستخدمون > طلبات المستخدمين**
2. عرض التسجيلات المعلقة:
   - اسم المستخدم
   - البريد الإلكتروني
   - تاريخ التسجيل
   - حالة الطلب

3. لكل طلب:
   - انقر لمراجعة
   - انقر على **"موافقة"** للتفعيل
   - انقر على **"رفض"** للرفض

### إرسال بريد التسجيل

أعد إرسال بريد الترحيب/التحقق:

1. اذهب إلى **المستخدمون > المستخدمون**
2. انقر على المستخدم
3. انقر على **"إرسال بريد"** أو **"إعادة إرسال التحقق"**
4. تم إرسال البريد الإلكتروني إلى المستخدم

## مراقبة المستخدمين الحاليين

### عرض المستخدمين المتصلين الآن

تتبع زوار الموقع النشطين:

**الموقع:** المستخدمون > المستخدمون الحاليون

يعرض:
- المستخدمون الحاليون المتصلون
- عدد الزوار الضيوف
- وقت آخر نشاط
- عنوان IP
- موقع تصفح

### مراقبة نشاط المستخدم

فهم سلوك المستخدم:

```
المستخدمون النشطون: 12
مسجلون: 8
مجهول: 4

النشاط الأخير:
- المستخدم1 - منشور المنتدى (منذ دقيقتين)
- المستخدم2 - تعليق (منذ 5 دقائق)
- المستخدم3 - عرض الصفحة (منذ 8 دقائق)
```

## تخصيص ملف تعريف المستخدم

### تفعيل ملفات تعريف المستخدمين

كوّن خيارات ملف تعريف المستخدم:

**Admin > النظام > التفضيلات > إعدادات المستخدم**

```
السماح بملفات تعريف المستخدمين: ☑ نعم
إظهار قائمة الأعضاء: ☑ نعم
يمكن للمستخدمين تحرير الملف الشخصي: ☑ نعم
إظهار صورة المستخدم: ☑ نعم
إظهار آخر اتصال: ☑ نعم
إظهار عنوان البريد الإلكتروني: ☐ نعم / ☑ لا
```

### حقول ملف تعريف

كوّن ما يمكن للمستخدمين إضافته إلى ملفاتهم الشخصية:

حقول الملف الشخصي المثالية:
- الاسم الحقيقي
- عنوان موقع الويب
- السيرة الذاتية
- الموقع
- الصورة الرمزية (صورة)
- التوقيع
- الاهتمامات
- روابط وسائل التواصل الاجتماعي

كوّن في إعدادات الوحدة.

## المصادقة

### تفعيل المصادقة الثنائية

خيار أمان معزز (إذا كان متاحاً):

**Admin > المستخدمون > الإعدادات**

```
المصادقة الثنائية: ☑ مفعلة

الطرق:
☑ البريد الإلكتروني
☑ الرسالة النصية
☑ تطبيق المصادقة
```

يجب على المستخدمين التحقق بطريقة ثانية.

### سياسة كلمة المرور

أفرض كلمات مرور قوية:

**Admin > النظام > التفضيلات > إعدادات المستخدم**

```
الحد الأدنى لطول كلمة المرور: 8 أحرف
تطلب أحرف كبيرة: ☑ نعم
تطلب أرقام: ☑ نعم
تطلب أحرف خاصة: ☑ نعم

انتهاء صلاحية كلمة المرور: 90 يوم
فرض التغيير عند تسجيل الدخول الأول: ☑ نعم
```

### محاولات تسجيل الدخول

منع هجمات القوة الغاشمة:

```
القفل بعد محاولات فاشلة: 5
مدة القفل: 15 دقيقة
تسجيل جميع المحاولات: ☑ نعم
إخطار المسؤول: ☑ نعم
```

## إدارة البريد الإلكتروني للمستخدم

### إرسال بريد جماعي إلى مجموعة

مراسلة المستخدمين المتعددين:

1. اذهب إلى **المستخدمون > المستخدمون**
2. حدد مستخدمين متعددين (خانات اختيار)
3. انقر على **"إرسال بريد"**
4. اكتب الرسالة:
   - الموضوع
   - نص الرسالة
   - أضف التوقيع
5. انقر على **"إرسال"**

### إعدادات تنبيهات البريد الإلكتروني

كوّن رسائل البريد الإلكتروني التي يتلقاها المستخدمون:

**Admin > النظام > التفضيلات > إعدادات البريد الإلكتروني**

```
التسجيل الجديد: ☑ إرسال بريد ترحيب
إعادة تعيين كلمة المرور: ☑ إرسال رابط إعادة التعيين
التعليقات: ☑ إخطار على الردود
الرسائل: ☑ إخطار الرسائل الجديدة
الإشعارات: ☑ إعلانات الموقع
التكرار: ☐ فوري / ☑ يومي / ☐ أسبوعي
```

## مهام إدارة المستخدمين الشائعة

### إنشاء مستخدم مسؤول

1. أنشئ مستخدماً جديداً (الخطوات أعلاه)
2. عيّن إلى مجموعة **صاحب الموقع** أو **مسؤولون**
3. امنح الأذونات في النظام > الأذونات
4. تحقق من أن وصول المسؤول يعمل

### إنشاء مشرف

1. أنشئ مستخدماً جديداً
2. عيّن إلى مجموعة **مشرفون**
3. كوّن الأذونات للإشراف على وحدات محددة
4. يمكن للمستخدم الموافقة على المحتوى وإدارة التعليقات

### إعداد محررو المحتوى

1. أنشئ مجموعة **محررو المحتوى**
2. أنشئ مستخدمين وعيّنهم إلى المجموعة
3. امنح الأذونات ل:
   - إنشاء/تحرير الصفحات
   - إنشاء/تحرير المنشورات
   - إشراف التعليقات
4. قيّد وصول لوحة المسؤول

### إعادة تعيين كلمة مرور منسية

نسي المستخدم كلمة مروره:

1. اذهب إلى **المستخدمون > المستخدمون**
2. ابحث عن المستخدم
3. انقر على اسم المستخدم
4. انقر على **"إعادة تعيين كلمة المرور"** أو تحرير حقل كلمة المرور
5. اضبط كلمة مرور مؤقتة
6. أخطر المستخدم (إرسال بريد)
7. يسجل المستخدم دخول ويغير كلمة المرور

### استيراد جماعي للمستخدمين

استيراد قائمة مستخدمين (متقدم):

توفر العديد من لوحات الاستضافة أدوات ل:
1. تحضير ملف CSV مع بيانات المستخدم
2. تحميل عبر لوحة المسؤول
3. إنشاء حسابات جماعية

أو استخدم السكريبت المخصص/الملحق للاستيرادات.

## خصوصية المستخدم

### احترم خصوصية المستخدم

أفضل ممارسات الخصوصية:

```
افعل:
✓ أخف رسائل البريد الإلكتروني افتراضياً
✓ اسمح للمستخدمين بالاختيار حول الرؤية
✓ حماية ضد البريد العشوائي

لا تفعل:
✗ مشاركة البيانات الخاصة
✗ عرض بدون إذن
✗ استخدم للتسويق بدون موافقة
```

### الامتثال لـ GDPR

إذا كنت تخدم مستخدمي الاتحاد الأوروبي:

1. احصل على الموافقة على جمع البيانات
2. اسمح للمستخدمين بتنزيل بياناتهم
3. توفر خيار حذف الحساب
4. حافظ على سياسة الخصوصية
5. سجل أنشطة معالجة البيانات

## استكشاف مشاكل المستخدمين

### لا يمكن للمستخدم تسجيل الدخول

**المشكلة:** نسي المستخدم كلمة المرور أو لا يمكنه الوصول إلى الحساب

**الحل:**
1. تحقق من أن حساب المستخدم "نشط"
2. إعادة تعيين كلمة المرور:
   - Admin > المستخدمون > ابحث عن المستخدم
   - اضبط كلمة مرور مؤقتة جديدة
   - أرسل إلى المستخدم عبر البريد الإلكتروني
3. امسح ملفات تعريف الارتباط/ذاكرة التخزين المؤقت للمتصفح
4. تحقق من عدم قفل الحساب

### تعليق التسجيل

**المشكلة:** لا يمكن للمستخدم إكمال التسجيل

**الحل:**
1. تحقق من السماح بالتسجيل:
   - Admin > النظام > التفضيلات > إعدادات المستخدم
   - تفعيل التسجيل
2. تحقق من أن إعدادات البريد الإلكتروني تعمل
3. إذا كان التحقق بالبريد الإلكتروني مطلوباً:
   - أعد إرسال بريد التحقق
   - تحقق من مجلد البريد العشوائي
4. قلل متطلبات كلمة المرور إذا كانت صارمة جداً

### حسابات مكررة

**المشكلة:** المستخدم لديه حسابات متعددة

**الحل:**
1. حدد الحسابات المكررة في قائمة المستخدمين
2. احتفظ بالحساب الأساسي
3. دمج البيانات إذا كان ممكناً
4. احذف الحسابات المكررة
5. فعّل "منع بريد إلكتروني مكرر" في الإعدادات

- [ ] اضبط خيارات صورة المستخدم
- [ ] اختبر عملية التسجيل
- [ ] أنشئ حسابات اختبار
