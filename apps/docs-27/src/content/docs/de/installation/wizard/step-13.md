Diese Seite erfasst Ihre Wahl für die Module, die auf Ihrer neuen Website installiert werden sollen. Module können jederzeit bei Bedarf hinzugefügt oder entfernt werden.

Wählen Sie nach Eingabe der angeforderten Informationen und Korrektur etwaiger Probleme die Schaltfläche "Weiter", um fortzufahren.

![XOOPS Installer Module installieren](/xoops-docs/2.7/img/installation/installer-13.png)

## In diesem Schritt erfasste Daten

### Verfügbare Module

Wählen Sie "Ja", um ein Modul zu installieren. Sie können kein, ein beliebiges oder alle Module auswählen.

Dies sind die Module, die in einer Standard-XOOPS-2.7.0-Installation enthalten sind.

* Private Messaging - _erweiterte private Nachrichten zwischen Benutzern_
* User Profile - _erweiterte/benutzerdefinierte Benutzerprofile_
* Protector - _erweiterte Website-Sicherheit, Spam-Steuerung und Angriffsmilderung_
* DebugBar - _eine Entwicklungs- und Staging-Debug-Symbolleiste basierend auf Symfony VarDumper; nützlich für Modulentwickler und zur Diagnose von Produktionsproblemen, aber normalerweise nicht auf einer öffentlichen Website erforderlich_

Das Modul _System_ wird immer installiert und wird hier nicht als Option aufgeführt.

#### Hinweis

Sie können Module zum Modulsverzeichnis hinzufügen, wenn Sie die XOOPS-Dateien vor dem Start der Installation hochladen, und sie werden auch in diesem Schritt verfügbar sein.

#### Starke Empfehlung

**Alle XOOPS-Websites** sollten das Modul _Protector_ installieren.
